# Interactive Paper Sheet Ratios - Project Instructions

**Version:** 1.0.0  
**Last Updated:** 16 April 2026

---

## Project Overview

This plugin provides an interactive canvas-based visualization showing the relationship between paper sizes. Users can adjust the dimensions of the smallest (datum) sheet and see the nested sizes reconfigure in real-time with visual feedback.

**Key Characteristics:**
- Shortcode-based (`[paper_sheet_ratios]`)
- Canvas rendering with auto-scaling
- Real-time user interaction via JavaScript
- No admin interface (pure front-end)
- Mobile-responsive design

## Architecture Decisions

### No Admin Interface
- Configuration via shortcode attributes only
- No Settings API needed
- No admin menu items
- Pure front-end user interaction

### Canvas Rendering Strategy
- **4 Layers** (bottom to top):
  1. Background grid (grey/checkered)
  2. Color overlay (red/green variable opacity)
  3. Paper rectangles (nested sizes)
  4. Labels and text

### Auto-Scaling Algorithm
- Calculate bounding box of largest (outer) sheet
- Add padding (default: 20mm)
- Scale to fit canvas container
- Grid scales with object for visual depth effect

### Real-Time Updates
- Debounced canvas redraws (100ms recommended)
- Bidirectional sync: numeric inputs ↔ sliders
- Immediate calculation updates
- Smooth opacity transitions

## Calculation Formulas

### Area Doubling (Each Level Up)
```
Area(level_n) = Area(datum) × 2^n

For 5 levels (A4→A0):
- Level 0 (A4): W × H
- Level 1 (A3): W × H × 2
- Level 2 (A2): W × H × 4
- Level 3 (A1): W × H × 8
- Level 4 (A0): W × H × 16
```

### Dimension Calculation
When doubling area while maintaining aspect ratio:
```
new_width  = old_width × √2  (if orienting horizontally)
new_height = old_height × √2 (if orienting vertically)
```

### Aspect Ratio
```
ratio = outer_width / outer_height
```
Display with configurable decimal precision.

### Logarithmic Opacity
```
deviation = |current_area - 960000|
max_deviation = 960000 (worst case: area = 0 or area = 1920000)
normalized = log(deviation + 1) / log(max_deviation + 1)
opacity = min(opacity_cap, normalized × opacity_cap)

Color:
- Red (area > 960000)
- Green (area < 960000)
- Transparent (area = 960000)
```
